Git Bash Commands

  1. General
    • cd xxdir 前往某个目录下
    • cd .. 返回上级目录
    • ls 列出目录里的文件
    • cat xxfile 显示xxfile里的内容
    • pwd 显示当前目录的路径
    • history显示所有用过的command
    • mv xxfile newnamefile 改文件名
    • rm -rf xxdiv 删除某个目录
    • touch xxfile 新增xx文件
    • mkdir xxname 创建新文件夹xxname
    • git mv xxfile xxfolder/ 把xxfile移动到xxfolder里
    • ../ ../../ link里的文件地址上层文件夹或上上层文件夹
  2. Repository
    • git clone xxURL 从GitHub上copy需要clone的url下载到本地
    • git config --get remote.origin.url 查看GitHub的url
    • git add xxfile 把xxfile更新到缓存区
    • git commit -m “some messages” 确认所有更新
    • git add --all && git commit -m "comment" 更新缓存一次性完成，包括所有文件
    • git pull 把GitHub上的更新下载到本地
    • git push 把本地的更新上传到GitHub上 git push -f 强制更新remote
  3. Branch
    • git checkout xxbranch 转到xxbranch上
    • git push -set-upstream origin xx branch 把本地的branch推送到GitHub上
    • git fetch + git merge = git pull
    • git merge xxbranch 把xxbranch和main的内容合并
    • git log 显示commit的数量和内容 q退出
    • git branch -d branch_name 删掉本地branch, 如果要强制删除就-D git push origin --delete branch_name删除远程branch
    • git push origin main branch_name 把main的commit推送到remote上的branch
    • git push origin main:main bid:bid 同时推送main和branch
  4. Flask
    • flask --app hello run 在python里运行flask app hello.py
    • flask --app hello run --debug
  5. Django
    • 新建程序步骤：

      django-admin startproject xxproject *自动建一个项目文件夹
      python manage.py runserver 运行server
      manage.py startapp xx 建一个新的app 'xx'
      找到setting.py, 把新的app'xx',添加到installed_apps里
      views.py 建一个新的def, 把return后面的功能（比如request，或者httpresponse）import到文件里 在对应app文件夹下建一个urls.py（注意一个project可以有几个app，每个app都可以有对应的urls文件）
      urls.py 里建新的urlpattern， 注意要import上一步创建好的views, .的意思是当前文件夹
      到上一级的文件夹里找总的urls.py文件，把上一步的urls文件include到urlpattern里，然后import include
    • 改css文件没反应用ctrl + F5略过缓存
    • run migrations after making changes to your models python manage.py migrate
    • title() 返回一个字符串的副本，其中所有单词的第一个字符都被大写，而字符串方法 capitalize() 返回一个字符串的副本，其中只有整个字符串的第一个单词被大写。
    • 用meidad的步骤：

      在django中启用media文件夹储存图片，首先在项目文件夹下面建media文件夹
      在setting.py里改1）TEMPLATES下'context_processors'加上'django.template.context_processors.media'
      2)加上MEDIA_URL = '/media/' MEDIA_ROOT = os.path.join(BASE_DIR, 'media')
      urls.py里加上urlpatterns = [ 。。。。 ] + static(settings.MEDIA_URL, document_root=settings.MEDIA_ROOT)
      最后template里url如下{{ MEDIA_URL }}items/{{ item_number }}/{{ image }}
